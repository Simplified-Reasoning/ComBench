ComBench

A benchmark for rigorous proof reasoning and constructive realization in Olympiad-level combinatorics.

ComBench evaluates large language models on 100 human-annotated competition-level combinatorics problems. It separates proof quality from explicit witness construction through rubric-guided judging and deterministic verifier-gated scoring.

Overview

ComBench targets two capabilities that are both central to constructive combinatorics: proving the mathematical claim and realizing the requested object as a complete, checkable witness.

Rigorous Proof Reasoning

Models must produce Olympiad-style arguments scored by item-specific 0/1/6/7 grading guidelines.

Constructive Realization

Construction-centric records require an explicit witness payload in the prescribed representation.

Verifier-Gated Scoring

Deterministic Python verifiers check construction payloads and calibrate high proof scores when a witness fails.

Benchmark Design

Problems are curated from major Olympiad-style competitions and grouped into stable combinatorial reasoning categories.

Distribution of ComBench categories across 100 problems.

Annotation pipeline

Each record contains source metadata, the original problem, a reference answer, and problem-specific grading guidelines. Construction-centric records additionally include a construction instruction, reference witness, verifier plan, deterministic verifier code, and semantic audit.

The executable verifier reads the submitted construction payload through the same raw-standard-input interface used for model submissions.

Protocol diagrams: annotation pipeline and verifier-gated scoring rule.

Error Analysis

Below-full-credit proof failures concentrate on missing core mechanisms and wrong mathematical targets.

Primary proof-error distribution over below-full-credit proof samples.

The dominant failure mode is Missing Core Mechanism, accounting for 41.2% of below-full-credit proof solutions. Wrong Mathematical Target accounts for another 20.0%, showing that many failures are mathematical rather than purely formatting related.
